Project Overview

Clean drinking water is fundamental to public health, yet data about water quality is often complex or hard to access. Our project leverages machine learning to bridge this gap.

The app allows users to input a US ZIP code to instantly retrieve recent measurements from the Water Quality Portal. We then calculate a Water Quality Index (WQI) score and use predictive models to estimate trends, helping communities better understand their environment.

Data Sources

Live data from USGS/WQP and training data from global Kaggle repositories.

Core Tech

Python, Streamlit, Scikit-learn (Random Forest), Pandas, Pytest.

The Goal

Educational awareness about water safety with transparent uncertainty.

Data Engineering Details

We focused heavily on data quality before modeling. Key steps included:

  • Standardization: Converting diverse nitrate units (e.g., mg{NO3}/L) into a unified mg/L as N standard.
  • Geocoding: Using pgeocode to map ZIP codes to latitude/longitude for spatial queries.
  • Sanitization: Filtering out physically impossible values (e.g., negative pH) to reduce noise.
  • Feature Creation: Engineered "pollution_stress" indicators and decade-based temporal features.

Example of temporal water quality trends analyzed.

Model & Evaluation

We utilized Random Forest algorithms for their robustness against overfitting and ability to handle non-linear relationships in chemical data.

98.3% Classifier Accuracy
0.968 Regressor R² Score
580 Test Samples
Note on Performance: These metrics reflect performance on the Kaggle global dataset holdout set. Performance on real-world US tap water data may differ due to domain shift and local geological variances.

Deployment & Monitoring Strategy

To ensure reliability, our models are serialized as .joblib files and loaded dynamically within the Streamlit runtime. Our "light-weight" monitoring plan includes:

  • Retraining Loop: Periodic re-evaluation of the model against new data to track drift in accuracy and R² over time.

Potential Benefits

  • Empowers local communities to ask better questions about their water.
  • Democratizes access to complex EPA/USGS data through a simple UI.
  • Serves as an educational tool for understanding chemical factors (pH, Turbidity) in water safety.

Risks & Limitations

  • Domain Shift: Training on global data may not perfectly represent specific US micro-climates.
  • Data Gaps: Rural or underserved regions often have fewer monitoring stations, leading to potential equity bias in prediction quality.
  • Simplification: WQI does not capture all contaminants like PFAS or heavy metals.

Disclaimer: This project is for educational purposes only. Do not use it to make real health, medical, or regulatory decisions.

Get Started

Run the project locally to explore the code and models.

1. Clone the repository

git clone https://github.com/seanesla/group11C

2. Install dependencies (using Poetry)

poetry install

3. Run the Streamlit App

poetry run streamlit run streamlit_app/app.py
